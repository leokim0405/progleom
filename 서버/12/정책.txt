계정 정책
  - 계정의 암호를 추측하여 공격하는 Brute Force 공격이나 특정 단어가 포함된 암호를 추측하여 공격하는 사전 공격을 완화

  - 암호 정책과 계정 잠금 정책으로 분류

  - 암호 정책 : 암호의 만료 기간과 복잡성 설정 등 암호 자체에 대한 보안 정책
    > 암호는 복잡성을 만족해야 한다	: 암호의 복잡도 설정 4가지 중 3가지 이상을 사용한 암호 사용하도록 설정
       최근 암호 기억			: 최근에 설정했던 암호를 몇 개나 기억 할 것인지 설정
       최대 암호 사용 기간		: 암호를 최대 몇 일 까지 사용 가능한지 설정
       최소 암호 길이			: 암호를 최소 몇 글자 이상 사용해야 하는지 설정
       최소 암호 사용 기간		: 암호 변경 후 변경한 암호를 유지해야 하는 기간 설정
       해독 가능한 암호화 사용		: 사용자의 암호를 다른 운영체제에서 해독 가능한 암호화를 사용해 저장

  - 계정 잠금 정책 : 비밀번호를 틀렸을 때 계정의 보호를 위한 계정 잠금과 관련된 보안 정책
    > 계정 잠금 기간			: 계정이 잠기고 난 후 몇 분 동안 잠금 상태를 유지할 것인지 설정
       계정 잠금 임계값		: 암호를 몇 번 틀려야 계정이 잠금처리 되는지 설정
       계정 잠금 수를 원래대로 설정	: 계정 잠금 임계값에 도달하기 전에 적용, 현재 암호를 틀린 회수를 몇 분 후에 초기화 할 것인지 결정

  - 계정 정책을 포함한 로컬 보안 정책의 비활성화 된 각종 항목들은 보안 템플릿을 사용하여 정책 변경이 가능

보안 템플릿
  - 사용자를 위해 정의되어 있는 보안 설정

  - 여러 컴퓨터에 공통된 보안 설정을 하기 위한 용도로 사용 가능

  - 그룹 정책으로 배포 가능

  - 보안 템플릿으로 정책을 설정하고 보안 구성 및 분석 콘솔을 이용하여 정책을 적용
