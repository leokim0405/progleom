백업
 - 자료의 보존을 위해 데이터의 복사본을 만드는 작업

 - 서버는 일정을 정해놓고 주기적으로 백업을 진행
  1. 백업 도구 선택
    1) 윈도우에서 제공하는 백업 도구
	CLI환경의 백업(명령줄 도구)

	섀도 복사본
	 > 볼륨 단위로 섀도 복사본 설정을 통해 진행
	   읽기 전용이기 때문에 백업 후 내용 확인 불가능
	   백업/복구가 볼륨 단위로만 이루어지기 때문에 개별 파일의 백업/복구 불가능
	   복구작업 수행 시 복구시점 이후에 만들어진 섀도 복사본 전부 사라진다
	   저장되는 섀도 복사본의 이름은 섀도 복사본이 생성된 날짜와 시간으로만 구성

	이전 버전
	 > 섀도 복사본을 이용한 기능
	   섀도 복사본의 단점을 어느정도 해소한 기능
	   (내부 내용 확인, 개별 복구, 복구 후 복구시점 이후 섀도 복사본 유지)

	윈도우 서버 백업
	 > 윈도우 서버에서 제공해주는 기능
	   섀도 복사본을 이용한 기능
	   이전 버전에 비해 더 사용하기 편한 인터페이스 제공
	   네트워크를 통해 다른 장치의 백업 진행이 가능

    2) 별도의 백업 프로그램을 구매(주로 사용)

  2. 백업 관리자 선정
    > 파일/폴더를 백업하기 위해서는 특정 접근 권한이나 사용자 권한이 필요
	Local computer : Administrators, Backup Operator
	Domain Controller : Administrators, Backup Operator

    > 하드 디스크의 사용을 제한하는 디스크 할당량 제한이 없어야 한다

    > 백업 관리자 권한을 없앨때는 백업 관리자의 계정을 백업 관련 그룹에서 제외
       + 물리적인 접근 권한도 없애줘야 한다(백업된 미디어가 보관된 장소에 출입을 금한다)

 - 백업 일정을 정해놓고 백업
    > 백업 일정이 정해지면 서버를 사용하는 사용자들에게 일정 안내

 - 백업에 필요한 비용을 산출할 수 있어야 한다
    > 소프트웨어(백업 프로그램)
      미디어(저장 장치)

 - 네트워크를 이용한 백업을 진행하는 경우에는 네트워크 대여폭을 확인하여 작업

 - 백업의 종류
    > 다섯가지 종류의 백업이 있다
	1. 매일 백업 : 자정부터 백업시점까지의 데이터를 백업, 특정 파일만 백업 X
	2. 보통 백업 : 모든 파일을 백업, 특정 파일만 백업 O (= 전체 백업)
	3. 복사 백업 : 모든 파일을 백업, 특정 파일만 백업 X
	4. 증분 백업 : 기존에 백업이 된 후로 변경된 데이터만 백업, 특정 파일만 백업 O
	5. 차등 백업 : 기존에 백업이 된 후로 변경된 데이터만 백업, 특정 파일만 백업 X

안전 모드
 - 최소한의 드라이브만 이용하여 부팅

 - 예전에는 드라이브 충돌 등의 오류로 인해 안전모드 사용
    > 요즘에는 잘 사용하지 않는다

 - 부팅 과정 중 F8 키를 입력하여 안전 모드에 접근 가능

 - 안전모드의 종류
	1. 안전 모드		   : 기본 파일과 드라이버만 사용하여 부팅
	2. 안전 모드(네트워킹 사용)	   : 기본 파일과 드라이버 및 네트워크 연결만 사용하여 부팅
	3. 안전 모드(명령 프롬프트 사용) : 기본 파일과 드라이버만 사용하여 부팅
				     부팅 후 바탕화면, 시작메뉴, 작업표시줄 등의 GUI화면이 아닌 명령 프롬프트만 출력

마지막으로 성공한 구성
 - 레지스트리가 수정되서 부팅이 불가능할때 사용
    > 이전에 부팅에 성공했던 레지스트리 값으로 재설정해서 부팅

 - 변경된 레지스트리 값을 수정할 수 있지만, 삭제된 레지스트리 값의 복구는 불가능

 - 이 기능을 정상적으로 활용하기 위해서는 windows를 정상 종료 시켜줘야 한다

IIS
 - Internet Information Service

 - Microsoft에서 개발한 windows 운영체제를 위한 웹 서버 프로그램

 - 사용자가 필요로 하는 기능/모듈들만 조합하여 가볍고 안정적인 웹 서버를 구성할 수 있다

 - windows server 운영체제에는 서버 관리자 - 역할 설치를 통해 구성 가능

 - 역할 서비스 구성에 따라 ftp 서버의 역할도 가능하다
