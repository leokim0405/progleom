공유 폴더 게시
 - 도메인 환경에서 사용
    > 공유 폴더를 Active Directory에 객체로 등록

 - 도메인에 가입된 컴퓨터들은 Active Directory에 게시된 공유 폴더를 검색하여 사용
    > 공유 폴더가 위치한 장치의 IP 주소 등의 정보를 알 필요가 없다

네트워크 드라이브 연결
 - 네트워크로 연결된 다른 컴퓨터의 폴더를 내 드라이브처럼 문자를 할당하여 사용

 - 해제 시 드라이브에 마우스 오른쪽 클릭하여 연결 해제 선택

오프라인 파일
 - 네트워크의 파일을 사용하던 중에 연결에 문제가 생기면 사용하던 파일을 내 컴퓨터에 복사
    > 네트워크를 통한 작업의 안정성 보장

 - 장점
	네트워크에 연결되지 않거나 연결 중 네트워크에 장애가 발생해도 계속해서 네트워크 파일에 접근이 가능
	네트워크 파일과 쉽게 동기화 할 수 있다
	느린 연결(네트워크 연결이 불안정한 경우)에서 작업하는 경우 효율적인 작업이 가능하다

   단점
	설정할 때 관리자 권한 필요
	원격 데스크톱과 같이 사용하면 충돌
	현재 오프라인 파일 쓸만한 환경이 없다

분산 파일 시스템
 - Distributed File System

 - 다수의 서버/장치에 분산된 공유 폴더들을 묶어서 내결함성을 제공하고 쉽게 접근할 수 있도록 해준다
    > 네임스페이스라는 폴더의 가상 트리에 공유 폴더를 배치한 후 사용자에게 제공

사용자 권한
 - User Right

 - 사용자, 그룹, 컴퓨터 계정이 시스템에 접근하거나 시스템을 제어할 수 있는 권한

 - 시스템에 영향을 주는 작업들을 일반 사용자나 특정 그룹이 할 수 있도록 만들어 주는 권한
