모니터링
 - 시스템에 문제가 있는지 지켜보는 작업
    > 갑작스러운 서비스의 실행 중단
      파일 시스템의 공간 부족
      응용 프로그램이 예외를 발생시켜 시스템 문제 발생

 - 모니터링은 로그 모니터링과 실시간 모니터링으로 구분
    > 로그 모니터링 : 모니터링 결과를 저장하고 나중에 확인
      실시간 모니터링 : 계속 지켜보는 모니터링

 - windows의 대표적인 모니터링 도구
    > 로그 모니터링 : 이벤트 뷰어, 데이터 집합기
      실시간 모니터링 : 작업 관리자, 성능 모니터

 - 서버를 모니터링 해야 하는 이유
    > 서버의 성능적인 문제를 해결
      서버 성능 개선

작업 관리자
 - 시스템 프로세스와 응용 프로그램, 하드웨어의 리소스 등을 관리하기 위한 도구

 - ctrl + shift + esc
   실행 -> taskmgr
   작업 표시줄에 마우스 오른쪽 클릭 -> 작업 관리자

성능 모니터
 - 카운터(성능 매개 변수)들의 통계 정보를 그래픽적으로 표시
    > 추적하고 있는 카운터들의 성능 정보를 보여주는 그래프를 생성

 - 결과를 로그 파일로 저장 가능

 - 실시간 모니터링 : 100초 동안의 동작
   데이터 수집기 : 성능 모니터의 결과를 로그로 저장

압축
 - NTFS 파일 시스템에서 기본적으로 제공하는 기능

 - 여러 파일/폴더를 하나로 묶지 않고 있는 그대로 압축

 - 기본적으로 압축 상태로 데이터를 보관
    > 사용자/시스템이 접근하면 자동으로 압축 해제하여 데이터 사용
    > 사용이 끝나면 다시 자동으로 압축

 - disk의 공간은 압축되기 전 파일 크기만큼 할당
    > 접근 시 자동으로 압축이 해제되기 위한 공간을 항시 확보

 - 속성값을 이용한 기능이기 때문에 상속 개념이 적용

 - 암호화와 동시 적용이 불가능하다

암호화
 - NTFS 파일 시스템에서 기본적으로 제공하는 기능

 - 기본적으로 데이터의 보호를 위해 데이터를 암호화
    > 사용자/시스템이 접근하면 자동으로 암호화 해제
    > 사용이 끝나면 다시 자동으로 암호화

 - 속성값을 이용한 기능이기 때문에 상속 개념이 적용

 - 압축과 동시 적용이 불가능하다

할당량
 - 용량을 제한하여 디스크를 효율적으로 사용하도록 만든 기능

 - 파일 서버 리소스 관리자 할당량(FSRM 할당량)과 NTFS 할당량으로 구분
