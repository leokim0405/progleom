디스크 관리
 - 하드 디스크의 물리적 구조
    > 플래터와 헤드로 구분
	플래터(platter)
	 실제 데이터가 저장되는 매체

	헤드(head)
	 플래터에 자료를 기록하거나 저장된 자료를 읽어오는 펜

    > 플래터는 여러장이 하나로 묶여서 모터에 연결되어 있고
      헤드는 헤드암에 연결되어 각 플래터마다 들어가있다

    > 디스크에 전원이 공급되면 플래터는 한 방향으로 회전하고
      헤드가 플래터에 데이터를 기록하고나 읽어온다

 - 하드 디스크의 논리적 구조
    > 트랙, 실린더, 섹터로 구분
	트랙(track)
	 플래터 전체 면적을 동심원을 기준으로 나눴을때 생기는 영역

	실린더(cylinder)
	 플래터 전체 면적을 구성하는 반지름 선

	섹터(sector)
	 트랙과 실린더가 교차되어 나오는 영역
	 실제 자료가 저장되는 단위(1 sector = 512 byte)

 - 디스크 관리 도구
    > GUI : 디스크 관리(diskmgmt.msc)
	로컬 컴퓨터나 원격 컴퓨터의 디스크 정보를 보거나 디스크를 관리할 수 있는 도구
	파티션/볼륨의 생성/삭제/확장/포맷, 드라이브 문자를 할당 등의 관리 작업

      CLI : diskpart
	GUI 환경보다 더 세밀한 명령이 가능
	더 많은(고급의) 작업을 수행하기 위해 사용
	메모장 파일을 이용하여 스크립트를 사용한 관리 가능

 - windows에서는 기본 디스크와 동적 디스크로 구분하여 디스크를 관리

기본 디스크
 - 파티션이라는 개념을 이용하여 디스크 용량을 분할하여 사용

 - 파티션 : 물리적인 디스크 하나를 여러개의 논리적인 디스크로 분할한 영역
	   > 연속된 섹터들을 묶어서 파티션을 생성

 - 파티션의 종류
    1. Primary partition
	주 파티션
	일반적으로 사용되는 파티션
	자료 저장 O, 운영체제 설치 O
	하나의 디스크에 최대 4개까지 생성(MBR) / 제한 없이 생성(GPT, windows에서는 128개로 제한)

    2. Extended partition
	확장 파티션
	자료 저장 공간(파티션)을 4개 넘게 사용하기 위해 생성(MBR)
	자료 저장 X, 운영체제 설치 X > Logical partition을 만들기 위한 영역
	(MBR)하나의 디스크에 단 1개만 생성

    3. Logical partition
	논리 파티션
	Extended partition을 분할하여 생성
	 > 하나의 디스크에서 모든 Logical partition의 용량의 합은 Extended partition의 용량을 넘을 수 없다
	   Extended partition이 없는 상태에서는 Logical partition 생성 불가
	자료 저장 O, 운영체제 설치 X > boot loader가 읽어오지 못하는 partition
	개수의 제한 X

 - MBR? GPT?
	MBR
	 > Master Boor Record, 지금까지 계속 사용하던 방식

	   주 파티션의 개수가 최대 4개까지 생성 가능(주 파티션 + 확장 파티션 합쳐서 최대 4개까지 사용)
	     * MBR 디스크는 디스크 초반 1MB를 MBR영역으로 사용하고 그 중 64byte 크기의 Partition Table 영역이 존재
		> Partition Table 영역에는 주 파티션/확장 파티션의 정보가 16byte 크기로 저장

	   하나의 파티션의 최대 크기 : 2.2TB

	   주소 체계 : CHS(Cylinder - Head - Sector)

	   BIOS 환경

	GPT
	 > GUID Partition Table, 점차적으로 도입되고 있는 방식

	   주 파티션의 개수 제한 X(단, windows 에서는 최대 128개로 제한)

	   하나의 파티션의 최대 크기 : 9.6ZB
	     * 용량 단위
		bit -> byte -> KB -> MB -> GB -> TB -> PB -> EB -> ZB -> YB
		    x8     x1024
		>>> 1ZB = 1024^3TB = 2^30TB(약 10억 7천만배)

	   주소 체계 : LBA(Logical Block Addressing)

	   EFI 시스템 사용

diskpart를 이용한 디스크 관리
 - 디스크, 파티션을 선택하여 명령어를 입력
    > 선택된 장치에 명령어가 적용

 1. 기본적인 디스크 관리 명령

  - 장치 확인
	list <장치종류>

    * 장치 종류
	disk	  : 디스크 목록 확인
	partition : 파티션 목록 확인(기본 디스크에서 사용)
	volume	  : 볼륨 목록 확인(동적 디스크에서 주로 사용)

  - 장치 선택
	select <장치 종류> <장치 번호>

  - 디스크 상세 정보 확인(디스크 선택 후 진행)
	detail disk

  - 디스크 추가 후 진행하는 작업(디스크 선택 후 진행)
	online disk	: 디스크를 온라인 상태로 전환
	attribute disk clear readonly
			: 디스크에서 읽기 전용 속성 제거

  - 디스크 전환
	convert <형태>

    * 형태 종류
	basic	: 기본 디스크(동적 디스크와 동시 사용 X)
	dynamic	: 동적 디스크(기본 디스크와 동시 사용 X)
	MBR	: MBR 형식(GPT와 동시 사용 X)
	GPT	: GPT 형식(MBR과 동시 사용 X)

 2. 기본 디스크
  - 특징
	하나의 디스크를 여러개의 파티션으로 분할
	 > 파티션은 연속된 섹터들의 집합

	4개의 주 파티션/3개의 주 파티션 + 1개의 확장 파티션 사용(MBR)
	주 파티션의 개수 제한 X(windows에서는 128개 제한) (GPT)

	여러 개의 물리적 디스크를 하나의 논리적 디스크로 구성 불가능

	디스크에 오류 발생 시 복구를 지원하지 않는다

  - 파티션 생성(파티션을 생성할 디스크 선택 후 진행)
	create partition <파티션 종류> size=<크기-MB단위>

    * 파티션 종류
	primary
	extended
	logical

    * 크기 생략 시 가능한 최대 용량으로 파티션 생성

  - 파티션 삭제(삭제할 파티션 선택 후 진행)
	delete partition
