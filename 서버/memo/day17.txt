diskpart를 이용한 디스크 관리
 1. 기본적인 디스크 관리 명령(추가)
  - 파티션(볼륨)의 확장/축소(파티션/볼륨 선택한 상태로 진행)

	확장 : extend size=<크기>

	축소 : shrink desired=<크기>

	 * 축소 전 축소 가능한 용량 확인
		shrink querymax

  - 포맷(파티션/볼륨 선택한 상태로 진행)
     > 포맷 작업은 파티션/볼륨 생성 후 파일 시스템 형식을 지정하기 위해 반드시 진행
       (windows에서는 NTFS가 기본, FAT32도 사용 가능)

     > xp/2003 까지는 format 기능이 diskpart에 없고 cmd에서 진행
       vista/2008 부터는 diskpart에서 format 지원

     > 사용 방법

	format fs=<FStype> [quick]

    * FStype
	ntfs
	fat32

    * quick을 붙여주면 빠른 포맷, quick을 안쓰면 전체 포맷


  - 장치 할당
     > 파티션/볼륨에 접근할 수 있는 기준을 생성하는 작업
       windows에서는 문자 할당/경로 할당(마운트) 작업을 통해 장치를 사용할 수 있다

     > 사용 방법(파티션/볼륨 선택한 상태로 진행)
	문자 할당	: assign letter=<문자>

	경로 할당	: assign mount=<폴더 경로>

	  * 문자 : A ~ Z 총 26개의 문자 중 사용 중인 문자를 제외하고 아무거나 입력
	    경로 : NTFS 장치 내부의 빈 폴더를 절대 경로로 입력

    * 문자 할당과 마운트는 동시에 사용 가능

 3. 동적 디스크
  - 특징
	하나의 디스크를 여러개의 볼륨으로 분할, 여러개의 디스크를 하나의 볼륨으로 생성
	 > 볼륨은 비 연속된 섹터들의 집합(연속된 섹터들을 하나의 볼륨으로 생성이 가능)

	볼륨의 개수 제한 X

	디스크에 오류 발생 시 복구를 지원하는 디스크 구조 생성 가능
	 > 기본적인 볼륨인 단순 볼륨부터 각종 디스크 구조를 생성하여 사용

	기본 디스크 -> 동적 디스크 전환은 파티션을 유지한 상태로 전환 가능(자료의 손실 없이 전환)
	동적 디스크 -> 기본 디스크 전환은 볼륨을 다 지우고 전환 가능(자료가 손실된다)

  - 단순 볼륨 생성

	create volume simple size=<크기>

    * 단순 볼륨이기 때문에 종류는 simple

    * 크기 생략 시 가능한 최대 용량으로 단순 볼륨 생성

  - 볼륨 삭제(삭제할 볼륨을 선택한 상태로 진행)

	delete volume

  - 스팬 볼륨(span volume)
	단순 볼륨을 다른 디스크로 확장하여 생성
	 > 단순 확장 외의 다른 기능은 없다

	동적 디스크 2개 이상 32개 이하까지 사용 가능

	내결함성을 갖지 않는다(오류 발생 시 데이터 보존 X)

	시스템 볼륨이나 부팅 볼륨은 스팬 볼륨으로 구성 X

	확장 및 축소 가능

     > 생성 방법(단순 볼륨 선택 후 진행)

	extend size=<크기> disk=<디스크 번호>

  - 스트라이프 볼륨(stripe volume)
	디스크 레이드의 일종(raid-0)
	 > 디스크 레이드는 여러개의 동적 디스크를 하나로 묶어서 각종 특수한 기능들을 수행하도록 만든 구조
	   windows에서는 0,1,5 3종류의 raid 지원

	속도가 장점인 raid 구조
	 > 여러 디스크에 데이터를 분할하여 동시 사용, 디스크가 늘어나는 만큼 속도가 빨라진다
	   단, 사용하는 디스크가 사양이 서로 다르면 가장 낮은 디스크의 사양에 맞춰진다

	동적 디스크 2개 이상 32개 이하 까지 raid-0으로 묶어줄 수 있다

	내결함성은 갖지 않는다

	확장 및 축소 불가능

     > 생성 방법

	create volume stripe size=<크기> disk=<디스크 번호>

 - 미러 볼륨(mirror volume)
	디스크 레이드의 일종(raid-1)

	자료를 저장하면 그 백업본을 함께 저장
	 > 단순 볼륨에 데이터가 저장되고 raid-1로 묶인 다른 볼륨에 동일한 내용이 저장

	동적 디스크 2개만 사용

	내결함성 제공
	 > 동일한 내용이 저장되어 있기 때문에 디스크 하나가 고장나면 다른 디스크에 자료 사용

	확장 및 축소 불가능

     > 생성 방법
      1) mirror volume 생성

	create volume mirror size=<크기> disk=<디스크 번호>

      2) 단순 볼륨 생성 -> mirror 추가(단순 볼륨 선택된 상태로 진행)

	add disk <디스크 번호>

  - Raid-5
	별도로 부여된 이름 없이 그냥 raid-5 라고 부른다

	디스크 여러개에 데이터를 분할

	추가 디스크 하나의 비용으로 내결함성 제공
	 > 짝수 패리티 비트를 이용하여 손실된 데이터를 복구

	동적 디스크 3개 이상 32개 까지 구성 가능

	확장 및 축소 불가능

     > 생성 방법

	create volume raid size=<크기> disk=<디스크 번호>
