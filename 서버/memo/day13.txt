 - 감사 이벤트 유형
	개체 액세스 감사		: 파일/디렉토리를 대상으로 하는 작업 전체를 감사(감사 지시)
	계정 관리 감사		: 계정 추가/변경/삭제 작업을 감사
	계정 로그온 감사		: 네트워크를 통한 로그온 작업 감사(단, 로그는 계정 로그온과 로그온 이벤트가 통합되어 저장)
	권한 사용 감사		: permission/user right에 대한 작업을 감사(사실상 user right 감사용)
	디렉토리 서비스 액세스 감사	: active directory에 접근하는 작업 감사
	로그온 이벤트 감사		: 로컬 로그온 작업 감사(단, 로그는 계정 로그온과 로그온 이벤트가 통합되어 저장)
	시스템 이벤트 감사		: 시스템 종료/부팅 작업에 대한 감사
	정책 변경 감사		: 정책을 변경하는 작업을 감사(이름에 정책 이라는 단어가 들어간 항목)
	프로세스 추적 감사		: 동작하는 모든 프로세스에 대한 감사(사용 X)
				   > 시스템을 사용하면서 너무 많은 프로세스가 실행/종료 되기 때문에 로그 용량이 너무 커진다

    * 위의 유형들을 제외한 감사는 윈도우에서 제공하지 않기 때문에 별도의 사용 프로그램(모니터링 tool)을 사용

 - 개체 액세스 감사의 경우 감사 지시 작업과 파일 지정 작업을 전부 해줘야 한다
    > 윈도우 전체에 파일/디렉토리가 너무 많기 때문에 감사를 할 파일/디렉토리를 지정
      (속성 - 보안 - 고급 - 감사 탭에서 파일 지정 작업 진행)

보안 로그
 - 감사를 진행하면 보안 로그가 남는다
    > 로그는 이벤트 뷰어 콘솔에서 확인 가능

 - 이벤트 뷰어 : 윈도우에서 기본으로 제공하는 모니터링 도구
	       > 로그가 남으면 그 로그를 확인하기 위해 사용

 - windows 로그 -> 보안 로그에 감사 기록이 저장된다(보안 로그 저장 위치 : C:\windows\systemc32\winevt\logs\security.evtx)
    > 필터링 작업을 통해 원하는 로그만 확인

 - 로그 동작(최대 로그 크기에 도달하면 수행할 작업)
    1. 필요한 경우 이벤트 덮어쓰기(가장 오래된 이벤트 먼저)
	> 로그 용량이 꽉 차면 가장 오래된 이벤트를 삭제하고 새로운 로그 저장

    2. 로그가 꽉 차면 로그 보관. 이벤트를 덮어쓰지 않음
	> 로그의 백업본을 만들고 새로 저장

    3. 이벤트 덮어쓰지 않음(수동으로 로그 지우기)
	> 로그가 꽉 차면 그대로 동작을 멈추고 로그 파일이 비워지면 다시 로그 기록을 실행)

관리자 권한으로 실행
 - windows XP / windows server 2003 까지는 '다음 계정으로 실행' 이라는 기능
    > 현재 로그온 한 사용자가 아닌 다른 사용자의 권한으로 특정 작업을 수행할 때 사용

 - 다른 사용자의 권한을 사용하는 일이 관리자 권한 말고는 거의 없어서 관리자 권한으로 실행 이라는 기능으로 변경
   (windows vista / windows server 2008 부터)

 - 서버 관리를 포함한 모든 작업을 수행할 때 관리자로 로그온하여 작업을 수행하는 것 보다
   일반 사용자로 로그온하여 필요할 때 관리자 권한을 사용하는 것이 보안상 더 안전하기 때문에 사용하는 기능

 - 사용 방법
	GUI	: 응용 프로그램을 마우스로 오른쪽 클릭 후 관리자 권한으로 실행 선택
	CLI	: runas 명령
