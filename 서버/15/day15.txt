컴퓨터 관리
 - 로컬/네트워크 컴퓨터를 관리하기 위한 기본 도구

 - 제한적인 관리
    > 각 드라이브 내부의 파일 관리 X
      실시간으로 발생하는 오류 확인 X

 - 콘솔 여는 방법 : 실행 -> compmgmt.msc

 - 네트워크 컴퓨터 관리 : 동작 -> 다른 컴퓨터에 연결

원격 컴퓨터 관리
 - 컴퓨터 관리의 다른 컴퓨터에 연결 기능과는 별개로 원격지의 컴퓨터를 세밀하게 제어하기 위해 사용

 - 장점
    원격 접속을 통해 효율적인 관리 작업 제공
    여러 문제를 빠르게 진단 가능
    네트워크로 연결만 가능하다면 어디서든 접근이 허용
    원격지에서 GUI 환경의 작업이 필요할 때 사용(원격 데스크톱, 원격 지원)

 - 종류
    1. 터미널 서비스
	원격 컴퓨터의 완벽한 관리를 위하여 사용
	별도의 터미널 서비스를 구성하여 제어
	잘 사용하지 않는 방식

    2. 원격 데스크톱
	windows 기본 기능
	터미널 서버가 설치되어 있지 않아도 사용 가능(단, 원격지의 컴퓨터에서 몇몇 설정 필요)
	동시 연결 수는 운영체제에 따라 달라진다(동시 연결 수에는 로컬 로그온이 포함)
	윈도우에서 방화벽 3389번 포트를 이용하여 연결 허용

	설정 옵션
	 1) 이 컴퓨터에 대한 원격 연결 허용 안 함
	     > 원격 데스크톱 접속 거부

	 2) 이 컴퓨터에 대한 원격 연결 허용
	     > 원격 데스크톱 접속 허용

	 * 네트워크 수준 인증을 사용하여 원격 데스크톱을 실행하는
	   컴퓨터에서만 연결 허용
	     > windows vista/windows server 2008 이상만 연결

	 * 네트워크 수준 인증(NLB)
	    > xp/2003 까지의 문제점 중 하나였던 네트워크 로그온 관련 이슈를 보완한 인증 방식
	      xp/2003 까지는 연결이 완료된 후 로그인을 시도 > 로그인이 실패해도 연결은 되어 있는 상태
	      vista/2008 부터는 네트워크에 접근할 때 부터 인증(로그인)을 시도 > 로그인에 실패하면 접속도 실패

    3. 원격 지원
	다른 사용자에게 도움을 요청하거나 내가 다른 사용자의 도움 요청을 받아 문제를 해결
	초대장을 만들어서 상대방에게 전달 > 초대장을 받은 사용자는 초대장을 통하여 연결 시도
	모든 세션이 암호화 되어 암호로 보호 > 보안성이 높다
